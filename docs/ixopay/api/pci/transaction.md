# Transaction
Process transactions and facilitate online payments.
info
The fields and constraints documented here reflect the Transaction API requirements. Depending on the adapter, additional fields may be required and field restrictions may be stricter. Always consult the relevant [Adapter](https://documentation.ixopay.com/adapters/simulator) documentation for adapter-specific requirements.
## [ Debit A Debit performs a complete customer initiated customer-to-merchant payment.](https://documentation.ixopay.com/api/pci/debit)## [ Preauthorize A Preauthorize reserves the payment amount on the customer's payment instrument.](https://documentation.ixopay.com/api/pci/preauthorize)## [ Incremental authorization An incremental authorization increases or prolongs the authorized amount on the customer's payment instrument.](https://documentation.ixopay.com/api/pci/incremental-authorization)## [ Capture A capture transaction completes the payment which was previously authorized with the [Preauthorize](./preauthorize) method.](https://documentation.ixopay.com/api/pci/capture)## [ Void A void transaction cancels a previously performed authorization made with the [Preauthorize](./preauthorize) method.](https://documentation.ixopay.com/api/pci/void)## [ Register A register transaction registers a customer's payment instrument for future use ([Debit](./debit) or [Preauthorize](./preauthorize)).](https://documentation.ixopay.com/api/pci/register)## [ Deregister A deregister transaction allows the deletion of a previously registered payment instrument.](https://documentation.ixopay.com/api/pci/deregister)## [ Refund A refund transaction reverses a payment which was previously performed with [Debit](./debit) or [Capture](./capture).](https://documentation.ixopay.com/api/pci/refund)## [ Payout A payout transaction credits the customer's account with the given amount.](https://documentation.ixopay.com/api/pci/payout)## [ Callback Receive status updates about transactions.](https://documentation.ixopay.com/api/pci/callback)
